## Align Packet: 무료체험 횟수 정책 변경 (1회 → 3회)

### 1. 당신이 요청한 것 (To-be)

**원문:** "PRD 1: 무료체험 횟수 정책 변경 (1회 → 3회). 변경 의도: 1회 제공 시 노쇼·튜터 노쇼 등으로 체험 기회가 소멸되어 결제 전환 모수가 줄어드는 문제를 해결한다. 3회 제공으로 서비스 가치를 충분히 체험할 기회를 확보하고, 결제 전환 유도 대상을 넓힌다. 포함 범위: 무료체험 총 3회 제공 규칙, 회차별 개방 조건(직전 회차 정상 종료 시 다음 회차 개방), 학생 노쇼·취소 시 1회 차감 규칙, 튜터 노쇼 시 복구 규칙, 3회 모두 소진 후 유료 체험레슨(5,000원/3회) 전환 규칙 및 차감 정책, 체험레슨 정상 완료 기준(기존 정책 준용), 레슨 시간 변경 정책(시작 2시간까지 무제한)"

**시스템이 해석한 방향:**
무료체험을 1회에서 3회로 확대하고, 회차별 순차 개방·차감·복구 규칙을 정의하며, 3회 소진 후 유료 체험레슨(5,000원/3회)으로 전환하는 정책·시스템·UI를 구현한다.

>
> 시나리오 2 — 학생 노쇼: 1회차를 예약했으나 10분 초과 미입장(노쇼)한다. 1회가 차감되어 잔여 2회가 된다. 노쇼도 '종료'로 처리되어 2회차가 개방된다.
>
> 시나리오 3 — 튜터 노쇼: 1회차 수업에서 튜터가 노쇼한다. 해당 회차가 즉시 복구되어 잔여 3회가 유지된다. 학생이 재예약하여 1회차를 다시 진행한다.
>
> 시나리오 4 — 학생 취소 (2시간 전): 1회차 예약 후 수업 시작 2시간 전에 취소한다. 회차가 차감되지 않고 잔여 3회가 유지된다. 학생이 재예약한다.
>
> 시나리오 5 — 학생 취소 (2시간 이내): 1회차 예약 후 수업 시작 1시간 전에 취소한다. 1회가 차감되어 잔여 2회가 된다. 2회차가 개방된다.
>
> 시나리오 6 — 유료 체험 전환: 무료 3회를 모두 소진한 학생이 '유료 체험레슨(5,000원/3회)'을 구매한다. 유료 체험에도 동일한 순차 개방·차감 정책이 적용된다.

### 2. 현재 현실 (As-is)

#### Experience 관점 — 지금 사용자가 보는 것

현재 체험 홈(Trial Home)은 단일 1회 체험 기준으로 설계되어 있습니다. StepTabs가 '신청→체험→리포트' 3단계를 표시하고, 체험 완료 후 정규 수강권 구매를 유도합니다. 노쇼 시 '체험 레슨권 없어졌어'라는 알림톡이 발송되며, 튜터 노쇼 시 '즉시 복구' 안내가 발송됩니다. 체험 유효기간(7일)이 하단에 표시됩니다.

#### Policy 관점 — 지금 적용되는 규칙

이용약관 제14조에서 노쇼(10분 초과 미입장)와 취소 규정을 정의하고 있으나, 체험권 전용 조항은 없습니다. 제16조에서 '체험권: 해당 없음'으로 일시정지 대상에서 제외하고 있습니다. 강사 노쇼 시 회차권은 보상 1회 추가, 무제한권은 2일 추가이며, 체험권 고유의 복구 규칙은 약관에 없습니다(현재 관행으로만 운영).

#### Code 관점 — 지금 시스템이 할 수 있는 것

백엔드 체험권은 `PODO_TRIAL` 타입으로 `nPurchased=1`이 하드코딩되어 있습니다. 수강권 시스템은 잔여 횟수(nPurchased - nUsed)로만 사용 가능 여부를 판단하며, '이전 레슨 완료 여부'에 따른 조건부 개방 로직이 없습니다. 유료 체험 상품 타입도 존재하지 않습니다.

**체험권 생성**: `TicketServiceV2Impl.java:305-341` — `addTrialTicket()`에서 `nPurchased=1`, `eventType=PODO_TRIAL`, `expireDate=가입일+7일`로 생성.

**활성 수강권 조회**: `TicketServiceV2Impl.java:122-169` — 우선순위: UNLIMIT/COUNT > PODO_TRIAL > BONUS. 잔여 횟수(`nPurchased - nUsed > 0`)와 만료일만 체크.

**온톨로지**: Ticket 엔티티에 순차 개방 관련 속성 없음. Lesson 상태: REGIST → RESERVED → START → FINISH (또는 CANCEL/NO_SHOW).

#### CST-001 | Code | 체험권 발급 수량 하드코딩 (nPurchased=1)

**이것이 무엇인가:**
체험권 발급 수량이 백엔드에서 1로 하드코딩되어 있습니다.

**왜 충돌하는가:**
PRD는 3회 제공을 요구하지만, 현재 addTrialTicket()이 nPurchased=1로 고정 생성합니다.

**처리하지 않으면:**
백엔드 TicketServiceV2Impl이 체험권을 1회만 발급하므로 3회 제공이 작동하지 않음

**변경 규모:** 백엔드 1개 메서드 수정 (TicketServiceV2Impl.addTrialTicket)

<details>
<summary>기술 상세 (Builder 참고용)</summary>

`TicketServiceV2Impl.java:305-341`에서 `nPurchased` 값을 1에서 3으로 변경하고, 기존 1회 체험 사용자와의 마이그레이션 방안을 결정해야 합니다. 기존 사용자 중 체험 미사용자(nUsed=0)에게 잔여 3회를 부여할지, 기사용자(nUsed=1)에게 추가 2회를 부여할지 결정이 필요합니다.
</details>

---

#### CST-002 | Code | 회차별 순차 개방 메커니즘 부재

**이것이 무엇인가:**
현재 수강권 시스템에는 '이전 레슨 완료 시 다음 회차 개방'이라는 순차 개방 메커니즘이 없습니다.

**왜 충돌하는가:**
PRD는 '직전 회차 정상 종료 시 다음 회차 개방'을 요구하지만, Ticket 엔티티는 잔여 횟수(nPurchased - nUsed)만으로 사용 가능 여부를 판단합니다. 순차 잠금/해제 상태를 저장하고 판별하는 로직이 없습니다.

**처리하지 않으면:**
3회를 동시에 개방하면 PRD의 '직전 회차 정상 종료 시 다음 회차 개방' 조건을 충족하지 못함

**변경 규모:** 백엔드 Ticket 또는 Lecture 도메인에 순차 개방 상태 필드 추가 + 개방 판정 로직 신규 구현

<details>
<summary>기술 상세 (Builder 참고용)</summary>

**선택지 1**: Ticket에 `currentOpenRound` 필드를 추가하여 개방된 회차 번호를 추적. 레슨 완료/노쇼 시 `currentOpenRound += 1`.

**선택지 2**: Lecture 완료 이벤트에서 동일 Ticket의 다음 회차 Lecture 생성을 트리거. 현재는 학생이 직접 Lecture를 생성하는 구조(`createPodoClassV3`)이므로, 수업 완료 콜백에서 다음 회차 생성 가능 여부 플래그를 세팅.

**선택지 3**: nPurchased=3이되 동시에 1개만 예약 가능하도록 예약 시 검증 로직 추가. 가장 단순하지만 '개방' 개념과는 다소 차이.
</details>

#### CST-003 | Policy | 학생 노쇼·취소 차감 범위와 '정상 종료' 정의 확정 필요

**이것이 무엇인가:**
학생 노쇼·취소 시 '1회 차감'의 범위와, 차감된 회차가 '정상 종료'로 간주되는지 여부가 미정의입니다.

**왜 충돌하는가:**
PRD는 '학생 노쇼·취소 시 1회 차감'을 명시했으나, 현행 정책은 노쇼와 2시간 이내 취소를 다르게 처리합니다(노쇼: 소진+패널티, 2시간 이내 취소: 소진만). 또한 차감된 회차가 '정상 종료'에 해당하는지에 따라 다음 회차 개방 여부가 달라집니다.

**처리하지 않으면:**
학생 노쇼 시 1회 차감은 PRD에 명시되어 있으나, '취소'와 '노쇼'의 경계, 그리고 노쇼로 차감된 회차가 순차 개방 조건의 '정상 종료'에 해당하는지 미정의. 이로 인해 다음 회차 개방 여부 판단 불가

**변경 규모:** 정책 판단 → 백엔드 조건 분기 로직 반영

**선택지:**
- 옵션 A: 노쇼·2시간 이내 취소 모두 '1회 차감 + 다음 회차 개방' (소진 = 종료로 간주)
- 옵션 B: 노쇼만 '1회 차감 + 다음 회차 개방', 취소는 '복구 후 재예약' (취소 ≠ 종료)
- 옵션 C: 노쇼·취소 모두 '1회 차감'하되 다음 회차는 개방하지 않음 (미사용 처리)

#### CST-004 | Policy | 튜터 노쇼 시 체험 회차 복구 방식 결정 필요

**이것이 무엇인가:**
튜터 노쇼 시 체험 회차를 어떤 방식으로 복구할지 결정되지 않았습니다.

**왜 충돌하는가:**
현행 관행은 '체험 레슨 즉시 복구(nUsed 원복)'이지만, 3회 순차 구조에서는 복구가 '해당 회차 번호를 재사용'하는 것인지, '별도 보상권을 추가 발급'하는 것인지에 따라 순차 개방 로직이 달라집니다.

**처리하지 않으면:**
현행 정책은 '체험 레슨 즉시 복구'이지만, 3회 순차 개방 구조에서 복구가 '해당 회차 재사용'인지 '별도 보상권 발급'인지 미정의. 복구 방식에 따라 순차 개방 로직 설계가 달라짐

**변경 규모:** 정책 판단 → 복구 로직 설계

**선택지:**
- 옵션 A: nUsed를 원복하여 해당 회차를 재사용 (현행 방식 유지, 순차 개방 상태도 원복)
- 옵션 B: 별도 보상 체험권 1회 추가 발급 (정규 수강권의 보상 정책과 동일하게 통일)

#### CST-005 | Code | 유료 체험레슨(5,000원/3회) 상품·결제 체계 부재

**이것이 무엇인가:**
무료 3회 소진 후 유료 체험레슨(5,000원/3회) 상품 타입과 결제 플로우가 시스템에 없습니다.

**왜 충돌하는가:**
현재 PODO_TRIAL은 무료 생성 전용이며 결제와 연결되지 않습니다. 유료 체험 상품을 구현하려면 새로운 상품 타입, 가격 정책, 결제 플로우, 그리고 기존 수강권 우선순위 체계와의 통합이 필요합니다.

**처리하지 않으면:**
기존 PODO_TRIAL 타입은 무료 전용이며 결제 플로우와 연결되지 않음. 유료 체험 상품 타입, 가격, 결제 플로우를 신규 구현하지 않으면 3회 소진 후 유료 전환 불가

**변경 규모:** 백엔드 상품·결제 도메인 + 프론트엔드 결제 UI 신규 구현

<details>
<summary>기술 상세 (Builder 참고용)</summary>

**필요 작업**:

**고려사항**: 유료 체험이 반복 구매 가능한지, 1회 한정인지에 따라 상품 설계가 달라집니다.

#### CST-006 | Experience | 체험 홈 UI의 회차 진행 표시 패턴 변경 필요

**이것이 무엇인가:**
체험 홈 UI의 StepTabs가 단일 체험(신청→체험→리포트)용으로 설계되어 있어 3회 진행 표시에 부적합합니다.

**왜 충돌하는가:**
3회 체험에서는 각 회차의 상태(잠금/진행 가능/완료)를 표시해야 하는데, 현재 StepTabs는 단일 진행 과정만 표현합니다. 또한 노쇼 알림톡의 '레슨권 없어졌어' 메시지도 3회 체험 문맥에 맞지 않습니다.

**처리하지 않으면:**
현재 Trial Home의 StepTabs(신청→체험→리포트)는 단일 체험 기준 설계. 3회 체험 진행 상태(1/3, 2/3, 3/3)와 회차별 잠금/해제 상태를 표현하는 UI 패턴이 없으면 사용자가 남은 체험 횟수와 다음 단계를 인지하지 못함

**변경 규모:** 프론트엔드 Trial Home 컴포넌트 + UX Writing 수정

<details>
<summary>기술 상세 (Builder 참고용)</summary>

**UI 변경 범위**:
- Trial Home StepTabs → 3회차 진행 표시 (예: ● ○ 🔒 형태)
- 각 회차 상태: 완료(●) / 진행 가능(○) / 잠금(🔒)
- 잔여 회차 텍스트: '무료 체험 N회 남음'
- 노쇼 알림톡: '체험 레슨권 없어졌어' → '체험 1회가 차감되었어. 남은 N회로 다시 예약해봐!'
- 체험 완료 후 CTA: '유료 체험 구매' 버튼 추가

**디자인 시스템 영향**: 기존 StepTabs 컴포넌트 확장 또는 새 컴포넌트 필요. Credit System의 3색 자산 분류(Section 7)와의 시각적 일관성 검토 필요.
</details>

#### CST-007 | Policy | 이용약관에 체험 관련 조항 추가 필요

**이것이 무엇인가:**
이용약관에 무료체험 횟수, 순차 개방, 차감, 유료 체험 전환 조항이 없습니다.

**왜 충돌하는가:**
체험 정책을 변경하면 소비자에게 고지해야 하는 법적 의무가 발생합니다. 현 약관은 체험을 수강권 일반 규정에 포함시키고 있어, 3회 제공·순차 개방·유료 전환 등 고유 규칙을 명시하지 않으면 분쟁 발생 시 근거가 불충분합니다.

**처리하지 않으면:**
현 약관에 무료체험 횟수, 차감 규칙, 유료 체험 전환 조항이 없음. 법적 고지 의무 미충족 시 소비자 분쟁 발생 가능

**변경 규모:** 약관 조항 추가 (제13조 또는 별도 체험 조항 신설)

**선택지:**
- 옵션 A: 제13조(수업 예약)에 체험 관련 하위 조항 추가
- 옵션 B: 별도 '제13조의2(체험레슨 정책)' 조항 신설
