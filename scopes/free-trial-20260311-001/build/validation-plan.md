# Validation Plan: 무료체험 횟수 정책 변경 (1회 → 3회)

scope: free-trial-20260311-001

---

### VAL-001 | CST-001 | inject

**검증 대상:** 체험 3회 허용 + 순차 개방
**검증 방법:** 1회차 예약 → 완료 → 2회차 예약 가능 확인. 미완료 상태에서 2회차 예약 시도 → 차단 확인.
**통과 조건:** 3회 순차 예약/완료 성공. 미완료 시 다음 회차 차단.
**실패 시 조치:** 순차 개방 로직 수정

**Edge cases:**

| 시나리오 | 예상 결과 |
|---------|----------|
| 1회차 노쇼 후 다음 회차 예약 시도 | 잔여 체험권이 있으면 예약 가능 (노쇼도 '종료'로 간주) |
| 3회 모두 완료 후 추가 예약 시도 | 잔여 0장 — 예약 차단 |

---

### VAL-002 | CST-002 | inject

**검증 대상:** 체험 수강권 nPurchased=3 + 첫 예약 시 발급
**검증 방법:** 첫 수업 예약 시 체험권 3장 발급 확인. 2회 예약 시 중복 발급 안 됨 확인.
**통과 조건:** 첫 예약 시 정확히 3장 발급. 이후 추가 발급 없음.
**실패 시 조치:** 중복 발급 방지 로직 수정

**Edge cases:**

| 시나리오 | 예상 결과 |
|---------|----------|
| 동시에 2개 예약 요청 | 1건만 처리, 중복 발급 방지 |

---

### VAL-003 | CST-003 | inject

**검증 대상:** 예약 시 3일 이내 수업만 선택 가능
**검증 방법:** 예약 시 4일 후 시간대 선택 시도 → 차단 확인. 3일 이내 시간대 → 허용 확인.
**통과 조건:** 3일 이내만 선택 가능.
**실패 시 조치:** 예약 시간 필터 수정

**Edge cases:**

| 시나리오 | 예상 결과 |
|---------|----------|
| 자정 직전 예약 시 3일 경계값 | 72시간 기준으로 정확히 계산 |

---

### VAL-004 | CST-004 | inject

**검증 대상:** 무료=TRIAL_FREE, 유료=TRIAL 결제 유형 분기
**검증 방법:** 무료 체험 결제 시 TRIAL_FREE 확인. 유료 체험 결제 시 TRIAL(카드 파킹) 확인.
**통과 조건:** 결제 유형 정확 분기.
**실패 시 조치:** PaymentType 분기 수정

**Edge cases:**

| 시나리오 | 예상 결과 |
|---------|----------|
| 무료 체험 소진 후 유료 체험 결제 | TRIAL 유형으로 처리 + 카드 정보 수집 |

---

### VAL-005 | CST-005 | inject

**검증 대상:** 노쇼 2장 소멸 + 2h전 취소 복구 + 튜터 노쇼 복구
**검증 방법:** 학생 노쇼 → nUsed +2 확인. 2시간 전 취소 → nUsed 변경 없음 확인. 튜터 노쇼 → nUsed 변경 없음 확인.
**통과 조건:** 각 시나리오별 잔여 체험권 수 정확.
**실패 시 조치:** 차감 로직 수정

**Edge cases:**

| 시나리오 | 예상 결과 |
|---------|----------|
| 체험권 1장 남은 상태에서 노쇼 (페널티 1장 부족) | 잔여 0장 (음수 방지). 추가 페널티 없음. |
| 2시간 경계값 (정확히 120분 전 취소) | 복구 처리 (2시간 전 포함) |

---

### VAL-006 | CST-006 | inject

**검증 대상:** 매회 체험 완료 시 AFTER_TRIAL 쿠폰 발급
**검증 방법:** 1회 완료 → 쿠폰 1장 발급 확인. 2회 완료 → 추가 1장 발급 확인. 3회 완료 → 추가 1장 발급 확인.
**통과 조건:** 3회 완료 시 쿠폰 3장 발급. 각 쿠폰 중복 사용 불가.
**실패 시 조치:** 쿠폰 발급 트리거 수정

**Edge cases:**

| 시나리오 | 예상 결과 |
|---------|----------|
| 노쇼로 종료된 회차에서 쿠폰 발급 시도 | 미발급 — DONE 상태만 트리거 |

---

### VAL-007 | CST-007 | inject

**검증 대상:** 정규 전환 시 잔여 체험 자동 취소
**검증 방법:** 체험 1회 완료 후 정규 결제 → 잔여 2회 자동 취소 확인.
**통과 조건:** 정규 결제 성공 + 체험 수강권 취소 + 체험 수업 CANCEL 전이.
**실패 시 조치:** cancelTrialTicket 로직 확인

**Edge cases:**

| 시나리오 | 예상 결과 |
|---------|----------|
| 체험 3회 모두 완료 후 정규 결제 | 취소할 잔여 체험 없음 — 정상 처리 |

### VAL-008 | CST-008 | inject

**검증 대상:** trialUsedCount/trialTotalCount 기반 상태 분기
**검증 방법:** API 응답에 remainingTrialCount 포함 확인. 프론트엔드 배너/리다이렉트/Greeting이 새 필드 기반으로 동작 확인.

**실패 시 조치:** API 응답 또는 프론트엔드 분기 수정

**Edge cases:**

| 시나리오 | 예상 결과 |
|---------|----------|
| 기존 사용자 (trialPaymentYn=Y, 새 필드 없음) 로그인 | 하위 호환 — 기존 필드로 fallback |
