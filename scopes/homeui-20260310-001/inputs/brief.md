## 변경 목적 (필수)
<!-- 왜 이 변경이 필요한가요? 해결하려는 문제를 설명해 주세요. -->

앱 홈 화면 — 상태별 UI 및 동작 구현 (PRD 4)
학생의 체험 진행 상태에 따라 홈 화면이 동적으로 변하여, 각 단계에서 다음 행동(신청 → AI학습 → 레슨 입장 → 결제)을 명확히 유도한다.

## 대상 사용자 (필수)
<!-- 이 변경이 영향을 미치는 사용자는 누구인가요? -->

무료체험을 진행 중인 학생 (미신청 ~ 체험 완료 ~ 결제 전환까지의 전 단계)

## 기대 결과 (필수)
<!-- 이 변경이 성공하면 어떤 상태가 되나요? -->

각 상태(미신청/예약완료/레슨입장가능/체험완료/노쇼·취소)별로 적절한 UI와 CTA가 표시되어, 사용자가 다음 행동으로 자연스럽게 유도된다.

## 포함 범위
<!-- 이번에 포함할 기능이나 변경 사항을 나열해 주세요. (선택) -->

공통 구조: 상단 GNB 우측("수강신청", "채팅상담"), 메인 문구 영역(사용자 이름 + 상태별 유도 문구, 1줄 16자 × 2줄), 프로그레스바(4단계).

상태 1 — 미신청: 메인 문구 + 프로그레스바 1단계 + 캐러셀 + 교재 미리보기·AI학습·60일 프로모션 버튼 + 하단 플로팅 "무료체험 신청하기". 앱 내 예약 완료 팝업(일시 + "교재 보기" 버튼).

상태 2 — 예약 완료, 레슨 대기: 메인 문구(AI학습 유도, 1회 이상 진입 시 동기부여 문구 5종 순환) + 프로그레스바 2단계 + 레슨 상태 카드(일시·커리큘럼·케밥 메뉴: 교재 변경 바텀시트, 시간 변경 바텀시트, 취소) + 수업 가이드·AI학습·출석 이벤트 + 하단 플로팅 "레슨 교재 보기".

상태 3 — 레슨 입장 가능(시작 10분 전~완료): 메인 문구 + 프로그레스바 3단계 + 하단 플로팅 "무료체험 입장하기". 튜터 노쇼 처리(+10분 미입장 시 안내 + 1회 복구 + 케어콜). 레슨 정상 완료 시 진단서 안내(카카오톡 플친 발송 + 앱 홈 확인).

상태 4 — 체험 완료, 결제 유도: 메인 문구 + 프로그레스바 삭제 + 첫 수강 혜택 카드(수강권 연장 혜택 + 결제 CTA + 케밥 메뉴 "무료체험 N회 남음" → 추가 체험 바텀시트) + 진단서 링크·쿠폰 확인·프로모션 배너 + 하단 플로팅 "포도 시작하고 혜택받기".

상태 5 — 튜터 노쇼 후: 1회 복구, 상태 1 화면으로 복귀.

상태 6 — 학생 노쇼·취소: 2회 차감 안내 문구 + 상태 1 신청 플로우 재진입.

3회 모두 소진(노쇼·취소 조합): 상태 4 화면 유지, 하단 플로팅 "5,000원으로 체험레슨 3회 더 받기"로 변경. 유료 체험 3회 종료 후 정규 결제 유도 화면 전환.

2·3회차: 각 회차 플로우는 1회차와 동일. 정상 종료 후 홈은 상태 4와 동일.
