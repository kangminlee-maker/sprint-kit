# 구독 도메인 정책 (Subscription Policies)

## 컨텍스트
이 문서는 구독 도메인의 비즈니스 정책을 상세히 기술합니다.

---

## 1. 결제 유형 정책 (paymentType)

### SUBSCRIBE (정기 결제)
- **정의**: 매월 자동으로 결제되는 정기 구독
- **결제 주기**: 1개월 단위
- **수강권 발급**: 매월 결제 성공 시 새 수강권 발급
- **해지 시점**: 다음 결제일 전까지 이용 가능

### LUMP_SUM (일괄 결제)
- **정의**: 전체 이용 기간을 한 번에 결제
- **결제 주기**: 3개월, 6개월, 12개월 등
- **수강권 발급**: 결제 즉시 전체 기간 수강권 일괄 발급
- **특이사항**: 월별 환산가 표시 (`총액 / 개월수`)

### TRIAL (체험)
- **정의**: 신규 사용자 체험용 단기 구독
- **가격**: 무료 또는 특별 할인가
- **이용 기간**: 보통 7일~14일
- **제한**: 1인 1회 체험 제한

### EXTEND (연장)
- **정의**: 기존 구독의 기간 연장
- **적용 조건**: 활성 구독이 있는 사용자만 신청 가능
- **결제**: 연장 기간에 대한 추가 결제
- **연장 대상 제외**: 마지막 정규 레슨권이 EXTEND인 경우 추가 연장 대상에서 제외 (D-0 ~ D-25 필터링 규칙)

### BONUS (보상권)
- **정의**: 운영팀이 수동으로 지급하는 보상 구독
- **결제**: 없음 (무료 지급)
- **용도**: 서비스 장애 보상, 마케팅 이벤트, CS 대응

### IPAD (현물 포함)
- **정의**: 아이패드 등 현물 포함 패키지 구독
- **특이사항**: 결제 금액에 현물 가격 포함

## 2. 구독 상태 정책 (status)

### 상태 값 정의
| 상태 | 설명 | 전환 조건 |
|------|------|----------|
| `SUBSCRIBE` | 정기결제 활성 | 정기결제 진행 중 |
| `LUMP_SUM` | 일괄결제 활성 | 일괄결제 후 이용 중 |
| `BONUS` | 보상권 활성 | 운영팀 보상권 지급 |
| `TRIAL` | 체험 중 | 체험 구독 진행 중 |
| `EXTEND` | 연장 중 | 연장 구독 이용 중 |
| `FINISH` | 정상 종료 | 이용 기간 만료 또는 해지 완료 |
| `DELETE` | 삭제됨 | 환불 또는 관리자 삭제 |
| `HOLD` | 일시 중지 | 결제 실패 등으로 일시 중지 |

### 상태 전이 규칙
```
SUBSCRIBE ──→ FINISH (해지 요청)
SUBSCRIBE ──→ HOLD (결제 실패)
HOLD ──→ SUBSCRIBE (재결제 성공)
HOLD ──→ DELETE (장기 미납)
```

## 3. 언어 유형 정책 (langType)

### 지원 언어
| 값 | 설명 | 수강권 발급 |
|-----|------|------------|
| `EN` | 영어 전용 | EN 수강권 1개 |
| `JP` | 일본어 전용 | JP 수강권 1개 |
| `ENJP` | 영어+일본어 | EN, JP 수강권 각 1개 |

### ENJP 수강권 분배 규칙
- 월 수업 횟수가 언어별로 **균등 분배**됨
- 예: 월 8회 ENJP 구독 → EN 4회 + JP 4회
- 무제한(999)의 경우 양쪽 모두 무제한

---

## 4. 커리큘럼 정책 (curriculumType)

### 커리큘럼 유형
| 값 | 설명 | 특징 |
|-----|------|-----|
| `BASIC` | 일반 회화 | 기본 영어/일본어 회화 |
| `BUSINESS` | 비즈니스 회화 | 비즈니스 상황 중심 |
| `SMART_TALK` | 스마트톡 | AI 기반 대화 훈련 |
| `TRIAL` | 체험용 | 체험 전용 커리큘럼 |

### 커리큘럼과 수강권 매칭
- 수강권은 구독의 `curriculumType`을 상속받음
- 수업 예약 시 수강권의 커리큘럼과 수업 유형이 일치해야 함
- 프리토킹은 BASIC, BUSINESS 모두 사용 가능

---

## 5. 수업 횟수 정책 (lessonCountPerMonth)

### 특수 값
| 값 | 의미 | 발급 수강권 |
|----|------|------------|
| `999` | 무제한 | eventType = "UNLIMIT" |
| `4, 8, 12...` | 월 N회 | eventType = "COUNT" |

### 횟수 계산 규칙
- ENJP 구독 시: `lessonCountPerMonth / 2` (언어별)
- 무제한(999)은 분배 없이 양쪽 모두 무제한

## 7. 가격 정책

### 가격 구성 요소
| 필드 | 설명 |
|------|------|
| `originPrice` | 정가 |
| `subPrice` | 판매가 (할인 적용) |
| `discountRate` | 할인율 (%) |
| `bookPrice` | 교재 가격 |
| `videoPrice` | 동영상 가격 |
| `lessonPrice` | 수업 가격 |
| `lessonVat` | 수업 부가세 |

### 일괄결제 가격 표시
```
총액 / 개월수 = 월 환산가
예: 360,000원 (12개월) → "360,000원 (월 30,000원)"
```

## 8. 일시정지 (Hold) 정책

### 일시정지 상태
- **상태 값**: `HOLD`
- **전환 조건**: 결제 실패 등으로 구독이 일시 중단
- **데이터 관리**: `UserHoldList` 엔티티로 관리

### 일시정지 유형
| holdType | 설명 |
|----------|------|
| `HOLDING` | 일시정지 중 |
| `PENALTY` | 페널티 (예: 2시간 이내 수업 취소) |

### 페널티 정책
**2시간 이내 수업 취소 페널티**
- **페널티 기간**: 72시간 (3일)
- **holdCode**: `CANCEL`
- **메시지**: "2시간 이내 수업취소로 인한 예약금지"
- **구현**: `UserHoldServiceImpl.java:91-107`
