## 변경 목적 (필수)
<!-- 왜 이 변경이 필요한가요? 해결하려는 문제를 설명해 주세요. -->

현재 포도스피킹은 "1인 1언어 1수강권" 제약이 있어, 기존 회원이 동일 언어 수강권을 추가 구매할 수 없다. 이로 인해:
1. 수강권 만료 전 미리 구매해두려는 고객의 니즈를 충족하지 못함
2. 기존 고객 대상 리텐션 프로모션(할인 쿠폰, 기간연장)을 자동화할 수단이 없음
3. 중복 구매 관련 CS 문의가 지속 발생

이 변경은 중복 구매 제약을 제거하고, 만료 임박 고객에게 단계별 리텐션 프로모션을 자동 발송하여 이탈을 방지하는 것이 목적이다.


## 대상 사용자 (필수)
<!-- 이 변경이 영향을 미치는 사용자는 누구인가요? -->

- 기존 활성 수강권을 보유한 포도스피킹 회원 전체 (만료 임박 고객이 1차 대상)
- 수강권을 복수 보유하게 될 고객 (비활성 수강권 관리, 활성화 순서 변경)
- 더블팩(영어+일본어) 사용자


## 기대 결과 (필수)
<!-- 이 변경이 성공하면 어떤 상태가 되나요? -->

- 기존 회원이 동일 언어 수강권을 무제한 중복 구매하여 비활성 상태로 보관 가능
- 만료 전 6단계 홈 팝업 + 4회 알림톡으로 자동 리텐션 프로모션 수신
- 세그먼트별 차등 할인 쿠폰 + 기간연장 혜택 제공
- 수강권 만료 전 리텐션율 +20%p 향상, 프로모션 전환율 10%+, CS 문의 80% 감소
- 비활성 수강권 활성화 순서를 드래그 앤 드롭으로 관리 가능
- 학습통계 리포트로 수강 성과 확인 및 재구매 동기 부여


## 포함 범위
<!-- 이번에 포함할 기능이나 변경 사항을 나열해 주세요. (선택) -->

- 수강권 중복 구매 제약 제거 (SMART_TALK 제외)
- 수강권 5상태 전이 (INACTIVE/ACTIVE/EXPIRED/DEPLETED/REFUNDED)
- 고객 주도형 비활성 수강권 활성화
- 기존 결제 페이지 UI + 프로모션 배너 (쿠폰/기간연장 상호배제)
- 결제 방식 분기 (같은 언어 ACTIVE 시 일괄결제만)
- 수강권 관리 UI (활성/비활성/만료 3섹션, 레슨권 기반)
- 활성화 순서 변경 (드래그 앤 드롭, 구매+보상+보너스 통합)
- 리텐션 홈 팝업 6단계 phase별 바텀시트
- 리텐션 프로모션 스케줄러 (D-25/D-14/D-7/H-12 알림톡)
- 세그먼트 기반 차등 할인 (4단계)
- 학습통계 리포트 (D-25 게이트, 실시간 우선)
- 마이포도 페이지 (레슨권 기반 + 레거시 분기)
- 환불 정책 (개별 수강권 단위)
- additional_days GT_SUBSCRIBE 하드매핑


## 제외 범위
<!-- 이번에 포함하지 않을 사항을 나열해 주세요. (선택) -->

- 다중 수강권 보유 시 예약/합산/차감 규칙 (현행 예약 시스템 유지, 별도 티켓)
- 홀딩(일시정지) 기능 변경
- 해지유도 페이지
- 앱 푸시/SMS 채널 (알림톡만 포함)
- 프로모션 랜딩 페이지 (결제 페이지 직접 진입으로 대체)
- 1개월 구독 수강권 (판매 중지)
- 갱신 중단 시점 안내


## 제약 및 참고사항
<!-- 이미 알고 있는 제약(일정, 기술 제한 등)이나 참고할 정보를 적어 주세요. (선택) -->

- 기존 PRD: https://github.com/re-speak/prd/blob/main/specs/duplicate-ticket-purchase-pg-1-real/planning-artifacts/prd.md (v3.8, 상세 FR/NFR/QA 포함)
- 기술 스택: Flutter 앱 + Spring Boot 백엔드 + JPA/QueryDSL + PortOne PG + 카카오 알림톡
- 레슨권 체인 구조: N개월 = N개 GT_CLASS_TICKET (순서 변경 시 날짜 재계산 필요)
- 구독(BILLING) 결제는 즉시 ACTIVE → 보관 불가 → 동일 언어 중복구매 시 일괄결제(LUMP_SUM)만 허용
- 연장레슨권 보유 유저는 레거시 화면 분기 (배포 시점부터 연장레슨권 판매 중단)
- 만료 기준 = 보너스 포함 전체 종료일(레슨권 체인 마지막 end_date)
- 프로토타입(React+MSW) 검증 완료: 7개 화면, 15 API 엔드포인트
- 개발자 미팅(2026-02-24) 결정사항 반영 완료
