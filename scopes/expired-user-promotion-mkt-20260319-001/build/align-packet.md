## Align Packet: 만료자 대상 프로모션 강화 및 메시지 테스트를 통한 rebuy 개선

### 1. 당신이 요청한 것 (To-be)

**원문:** "만료자 대상 프로모션 강화 및 메시지 테스트를 통한 rebuy 개선"

**시스템이 해석한 방향:**
LOW/MEDIUM 세그먼트 만료자(D+4 이후)의 재구매율을 향상시킨다. 현재 풀페이지 팝업의 "신규기능 런칭" 메시지를 "손실 회피 + 가격 강조" 메시지로 교체하고, 최종 결제 전환율로 성과를 측정한다. 현재 재구매율 2.21%를 개선하는 것이 목표.

**제안된 범위:**

| 포함 | 동의하시나요? |
|------|-------------|
| 풀페이지 팝업 헤드라인 변경: "지금 역대 최대 할인가로 다시 시작하세요!" | |
| 풀페이지 팝업 서브카피 변경: "포도와 열심히 쌓은 외국어 실력이 사라지고 있어요ㅠㅠ" | |
| 팝업 내 이미지 삽입: 월 4만원대가 잘 보이는 이미지 (CTA 위) | |
| CTA 변경: "4만원대로 다시 시작 >" → 세그먼트별 외부 프로모션 URL로 랜딩 | |
| 닫기 문구 변경: "할인 혜택 포기하기" | |

| 제외 | 동의하시나요? |
|------|-------------|
| 앱 밖 채널 (플친/LMS) 메시지 — 마케팅팀 별도 운영 | |
| 세그먼트 분류 로직 (LOW/MEDIUM) — 기존 유지 | |
| 쿠폰 금액/종류 — 기존 세그먼트별 금액 그대로 | |
| 팝업 노출 조건 (D+4, 당일 미노출, 활성 수강권 보유 시 종료) — 기존 로직 유지 | |
| 외부 프로모션 URL/랜딩페이지 — 이미 존재, 변경 없음 | |

**시나리오:**

> 만료자(D+4 이후)가 앱을 열면 풀페이지 팝업이 자동 노출된다. 헤드라인에서 "역대 최대 할인가"를 인지하고, 서브카피에서 "외국어 실력 사라짐"에 대한 손실 회피 동기를 느낀다. 월 4만원대 이미지를 확인하고, CTA를 탭하면 세그먼트별 외부 프로모션 URL로 이동하여 결제한다.
>
> 만료자가 팝업에서 "할인 혜택 포기하기"를 탭하면 당일은 팝업이 노출되지 않는다. 다음날 앱을 열면 다시 팝업이 노출된다.
>
> 만료자가 재구매하여 활성 수강권을 보유하게 되면 팝업이 더 이상 노출되지 않는다.
>

### 2. 현재 현실 (As-is)

#### Experience 관점 — 지금 사용자가 보는 것

만료자(D+4 이후)가 앱을 열면 풀페이지 팝업이 자동 노출된다. 현재 메시지는 "신규기능 런칭" 내용으로, 결제를 유도하고 있다. 닫기 시 당일 미노출, 다음날 재노출. 활성 수강권 보유 시 팝업 종료. 현재 재구매율 2.21%.

#### Policy 관점 — 지금 적용되는 규칙

"역대 최대 할인가" 표현은 사실 — 이용 시점까지 개인별 가장 큰 할인 혜택(PO 확인). 쿠폰 금액은 세그먼트별 기존 금액 유지. 매일 팝업 노출은 업계 관행에 부합(경쟁사 동일, PO 확인).

#### Code 관점 — 지금 시스템이 할 수 있는 것

풀페이지 팝업은 서버에서 컨텐츠 교체 가능(PO 확인). 닫기 시 당일 미노출 로직은 기존 구현 존재. WelcomeBackPromotionDto가 세그먼트(LOW/MEDIUM) 판별 후 MarketingController(GET /welcomeback-promotion)를 통해 프로모션 정보 제공.

MarketingController.java → WelcomeBackPromotionDto (세그먼트 판별). CTA 탭 시 세그먼트별 외부 프로모션 URL로 랜딩. 기존 노출 조건/종료 로직 재사용.

#### CST-001 | Experience | 팝업의 "월 4만원대" 표시와 실제 결제 금액이 세그먼트/쿠폰에 따라 다를 수 있어 사용자 혼란 가능

**이것이 무엇인가:**
팝업의 "월 4만원대" 표시와 실제 결제 금액이 세그먼트/쿠폰에 따라 다를 수 있음

**왜 충돌하는가:**
LOW와 MEDIUM 세그먼트의 쿠폰 금액이 다르므로, 모든 사용자에게 "4만원대"가 정확하지 않을 수 있음

**처리하지 않으면:**
팝업에서 본 가격과 랜딩페이지 가격이 다르면 신뢰 저하 및 이탈 가능

**변경 규모:** 사용자 신뢰에 영향. 가격 불일치 시 이탈 가능.

**선택지:**

| 선택 | 이점 | 리스크 | 내용 |
|------|------|--------|------|
| 모든 세그먼트에서 4만원대가 되는지 확인 후 진행 | 정확한 정보 제공 | 확인에 시간 소요 |  |
| "월 4만원대~" 등 범위 표현으로 변경 | 세그먼트 차이 커버 | 후킹력 약화 |  |

**추천:** 모든 세그먼트에서 4만원대가 되는지 먼저 확인

---

#### CST-002 | Policy | "역대 최대 할인가" 문구가 광고/프로모션 표시 규정에 부합하는지 확인 필요

**이것이 무엇인가:**
"역대 최대 할인가" 문구의 광고 표시 적정성

**왜 충돌하는가:**
소비자보호법상 "최대" 표현은 객관적 근거가 필요

**처리하지 않으면:**
과장 광고로 해석될 경우 소비자보호법 이슈 가능. 다만 개인별 최대 할인이 사실이므로 리스크 낮음.

**변경 규모:** 리스크 낮음 — 개인별 최대 할인이 사실이므로 근거 있음

**추천:** 현재 표현 유지. 단, 개인별 최대 할인임을 내부 기록으로 남겨둘 것을 권장

---

#### CST-003 | Code | 풀페이지 팝업의 헤드라인/서브카피/이미지/CTA를 서버에서 교체할 수 있는 구조가 필요

**이것이 무엇인가:**
팝업 컨텐츠 서버 교체 구조

**왜 충돌하는가:**
향후 메시지 테스트 반복을 위해 서버 제어가 필수

**처리하지 않으면:**
서버 교체 불가 시 매번 앱 업데이트가 필요하여 메시지 테스트 반복이 어려움

**변경 규모:** PO가 가능하다고 확인. 구현 상세는 빌드 단계에서 결정.

**추천:** Builder가 구현 방식 결정
