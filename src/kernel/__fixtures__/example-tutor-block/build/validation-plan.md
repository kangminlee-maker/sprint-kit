# Validation Plan: 튜터 차단 기능

scope: SC-2026-001
build_spec_hash: hash_bs_001

---

### VAL-001 | CST-001 | inject

**검증 대상:** 매칭 시 차단된 튜터 제외
**검증 방법:** 차단 목록에 튜터 B를 추가한 후 매칭 요청. 튜터 B가 결과에 포함되지 않아야 함.
**통과 조건:** 차단된 튜터가 매칭 결과에 0건
**실패 시 조치:** 구현 오류. apply 재시도

### VAL-002 | CST-002 | inject

**검증 대상:** 이용약관 예외 조항 추가
**검증 방법:** terms-of-service.md 제12조에 학생 선호 설정 예외 단서가 포함되어 있는지 확인
**통과 조건:** 예외 조항 텍스트 존재
**실패 시 조치:** 구현 누락. apply 재시도

### VAL-003 | CST-003 | inject

**검증 대상:** 프로필 화면 버튼 간격 16px 이상
**검증 방법:** 튜터 프로필 화면에서 차단 버튼과 인접 버튼 간 간격 측정
**통과 조건:** 간격 >= 16px
**실패 시 조치:** 구현 오류. apply 재시도

### VAL-004 | CST-004 | defer

**검증 대상:** 차단 관리 메뉴 순서 비변경
**검증 방법:** 설정 메뉴에서 차단 관리 항목의 위치가 기존과 동일한지 확인. 검증 파일: `figma:settings-menu`
**통과 조건:** 차단 관리 메뉴 위치 변경 없음
**실패 시 조치:** 의도하지 않은 간섭. `constraints_resolved`로 복귀

### VAL-005 | CST-005 | inject

**검증 대상:** 차단 기록 DB 테이블 생성
**검증 방법:** 차단 생성 후 DB에 레코드가 존재하는지 확인. 앱 재실행 후에도 차단 상태 유지 확인.
**통과 조건:** 차단 레코드 존재 + 재실행 후 유지
**실패 시 조치:** 구현 오류. apply 재시도

### VAL-006 | CST-006 | inject

**검증 대상:** 매칭 API 하위 호환성
**검증 방법:** 기존 앱 버전에서 매칭 API 응답을 정상 처리하는지 확인. 검증 파일: `api/v1/matching.ts`
**통과 조건:** 기존 앱에서 오류 없이 매칭 결과 표시
**실패 시 조치:** API 호환성 파괴. `constraints_resolved`로 복귀

### VAL-007 | CST-007 | inject

**검증 방법:** 3년 이상 경과한 테스트 차단 레코드를 생성한 뒤 배치 작업 실행. 해당 레코드 삭제 확인.
**통과 조건:** 경과 레코드 0건
**실패 시 조치:** 구현 오류. apply 재시도
